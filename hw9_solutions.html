1. שימוש ב-filter על רשימת מספרים

צור רשימה של 50 מספרים אקראיים בין 1-100 והדפס אותה

כעת השתמש ב filter וב- lambda כדי לקבל מתוך הרשימה:

a. רק מספרים הגדולים מ- 50

import random

numbers = [random.randint(1, 100) for _ in range(50)]

print(numbers)

greater_than_50 = list(filter(lambda x: x > 50, numbers))

print(greater_than_50)

b. רק מספרים המתחלקים ב-7 ללא שארית

print(divisible_by_7)

c. רק מספרים דו-ספרתיים (רמז: 10-99)

e. רק מספרים שסכום הספרות שלהם הוא 9

i.* בונוס: קלוט מהמשתמש 5 מספרים והכנס אותם לרשימה. כעת שמור ברשימה רק מספרים השונים מהמספרים שהוכנסו לרשימת המשתמש. רמז: in not

j.*** בונוס: רק מספרים ראשוניים. רמז: כתוב פונקציה המקבלת מספר ומחזירה אמת אם הוא ראשוני ושקר אם לא. כעת קרא לפונקציה שכתבת מתוך ה-lambda

def is_prime(n):

 if n <= 1:

  return False

for i in range(2, int(n**0.5) + 1):

  if n % i == 0:

   return False

 return True

prime_numbers = list(filter(lambda x: is_prime(x), numbers))

print(prime_numbers)

6. המשמעות של המילה global בתוך פונקציה

מה החסרון בשימוש ב-global כחלק מפונקציה? מה זה ידרוש בעתיד... מדוע כאן נקבל שגיאה:

x: int = 1

def foo():

print(x)

x = 4

foo()

המשמעות של המילה global בתוך פונקציה היא שהמשתנה שמוגדר כ-global הוא משתנה גלובלי שנמצא מחוץ לפונקציה, כך שכל השינויים בו בתוך הפונקציה ישפיעו גם מחוץ לה. החסרון בשימוש ב-global כחלק מפונקציה הוא שהדבר יכול להוביל לשגיאות וקונפליקטים בקוד, מכיוון שכל שינוי במשתנה הגלובלי יכול להשפיע על חלקים אחרים בקוד שמשתמשים באותו משתנה.

בדוגמה לעיל, השגיאה מתרחשת מכיוון שהפונקציה מנסה להדפיס את המשתנה x לפני שהוא הוגדר בתוך הפונקציה. הפונקציה מגדירה את x כמשתנה מקומי ולא כמשתנה גלובלי, ולכן השגיאה מתרחשת.
